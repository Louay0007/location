\section{Résumé Exécutif}

\subsection{Introduction}

Notre entreprise vous propose le développement d'une \textbf{plateforme web complète de gestion et réservation de véhicules}, conçue spécifiquement pour les agences de location mono-site. Cette solution moderne, robuste et évolutive combine les technologies les plus fiables du marché : \textbf{NestJS} pour le backend, \textbf{Angular} pour le frontend, et \textbf{PostgreSQL} pour la base de données.

\subsection{Pourquoi ce projet ?}

Le marché de la location de voitures en Tunisie connaît une transformation digitale accélérée. Les clients exigent désormais :
\begin{itemize}[leftmargin=*,noitemsep]
  \item Réservation en ligne en temps réel (24/7)
  \item Paiement électronique sécurisé (cartes, mobile money)
  \item Expérience utilisateur fluide sur mobile et desktop
  \item Accès à l'historique, contrats PDF, gestion de profil
  \item Transparence sur les disponibilités et tarifs
\end{itemize}

Sans présence en ligne structurée, votre agence risque de perdre des parts de marché au profit de concurrents déjà digitalisés.

\subsection{Notre engagement}

Nous livrons une solution \textbf{clé en main} comprenant :
\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Catalogue en ligne} avec filtres avancés (catégorie, carburant, prix, dates)
  \item \textbf{Système de réservation} intelligent avec vérification de disponibilité en temps réel
  \item \textbf{Paiements sécurisés} intégrant Click to Pay (Tunisie) et Stripe (international)
  \item \textbf{Back-office complet} pour gérer flotte, clients, réservations, finances
  \item \textbf{Génération automatique} de contrats PDF
  \item \textbf{Notifications emails} (confirmation, rappels, statuts)
  \item \textbf{Dashboard analytique} avec KPIs temps réel
\end{itemize}

\section{Contexte et Problématique}

\subsection{Analyse du besoin}

Votre agence gère actuellement un parc de \textbf{~50 véhicules} avec des processus majoritairement manuels :
\begin{itemize}[leftmargin=*,noitemsep]
  \item Réservations par téléphone / email / réception physique
  \item Suivi des véhicules sur feuilles Excel ou registres papier
  \item Paiements en espèces principalement (risques de sécurité, traçabilité limitée)
  \item Gestion des contrats manuelle (impression, archivage)
  \item Aucune visibilité en ligne → absence de réservations spontanées
  \item Difficulté à anticiper les tendances (saisonnalité, véhicules populaires)
\end{itemize}

\subsection{Conséquences}
\begin{itemize}[leftmargin=*,noitemsep]
  \item Perte de clients préférant la commodité du digital
  \item Charge administrative élevée (saisie manuelle double)
  \item Risques d'erreurs humaines (doublons, erreurs de disponibilité)
  \item Liquidités transactionnelles limitées
  \item Difficulté à concurrencer les agences nationales avec plateformes en ligne
\end{itemize}

\subsection{Solution proposée}

Développement d'une \textbf{plateforme web responsive} (mobile-first) permettant :
\begin{itemize}[leftmargin=*,noitemsep]
  \item Aux \textbf{clients finaux} : parcourir le catalogue, réserver en ligne, payer électroniquement, télécharger contrats, gérer historique
  \item Aux \textbf{administrateurs} : gérer flotte, tarifs, maintenance, dashboard analytique, reporting
  \item Aux \textbf{opérateurs} : traiter réservations, suivre paiements, imprimer contrats
\end{itemize}

  \item \textbf{JWT} : Authentification sans état (stateless)
  \item \textbf{Redis} : Cache sessions, rate limiting
  \item \textbf{Swagger} : Documentation API interactive (\texttt{/api/docs})

\subsubsection*{Frontend : Angular 17}
\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Angular Material} : Composants UI Material Design
  \item \textbf{NgRx} : Gestion d'état centralisée
  \item \textbf{Chart.js} : Graphiques dashboard admin
  \item \textbf{ngx-stripe} : Intégration paiements Stripe Elements
  \item \textbf{Responsive design} : Mobile-first, breakpoints 375px/768px/1024px/1440px
\end{itemize}

\subsubsection*{Infrastructure}
\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Nginx} : Reverse proxy, compression GZIP, SSL termination
  \item \textbf{Let's Encrypt} : Certificats SSL gratuits (HTTPS obligatoire)
  \item \textbf{PM2} : Process manager (auto-restart, logs)
  \item \textbf{Ubuntu 22 LTS} : Système d'exploitation serveur
\end{itemize}

\section{Modules Fonctionnels Détailés}

\subsection{Catalogue Véhicules (Public)}

\begin{itemize}[leftmargin=*,noitemsep]
  \item Affichage paginé (12 véhicules/page) avec filtres dynamiques :
    \begin{itemize}[leftmargin=*,noitemsep]
      \ Catégorie (Économie, Compact, SUV, Luxe...)
      \ Carburant (Essence, Diesel, Hybride, Électrique)
      \ Transmission (Manuelle/Automatique)
      \ Prix quotidien (min/max)
      \ Nombre de places
      \ Disponibilités par dates (calendrier)
    \end{itemize}
  \item Tri : prix croissant/décroissant, nouveauté, popularité
  \item Fiche détaillée par véhicule (photos, caractéristiques, avis clients)
  \item Indicateur de disponibilité en temps réel
  \item Calcul automatique du prix selon durée et règles tarifaires
\end{itemize}

\subsection{Système de Réservation}

Flux client en 4 étapes :
\begin{enumerate}[leftmargin=*,noitemsep]
  \item \textbf{Sélection véhicule + dates} : affichage du prix estimé
  \item \textbf{Informations personnelles} : formulaire client (CIN permis, coordonnées)
  \item \textbf{Choix paiement} : Click to Pay (Tunisie) ou Stripe (international)
  \item \textbf{Confirmation} : génération contrat PDF + email de confirmation
\end{enumerate}

Règles métier appliquées :
\begin{itemize}[leftmargin=*,noitemsep]
  \item Vérification conflit de dates (pas de chevauchements)
  \item Statut véhicule obligatoire (disponible, pas en maintenance)
  \item Date de début dans le futur (pas de réservation passée)
  \item Calcul du prix : tarif journalier × durée + cautions – réductions éventuelles
  \item Application automatique des règles tarifaires (saisonnières, weekend, long terme)
\end{itemize}

\subsection{Gestion des Paiements}

\subsubsection*{Click to Pay (Banque Centrale de Tunisie)}
\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Solution officielle} : Plateforme de paiement en ligne régulée par la Banque Centrale de Tunisie
  \item Intégration via \textbf{API REST Click to Pay} avec redirection sécurisée vers portail bancaire
  \item Accepte toutes les cartes bancaires tunisiennes (Visa, Mastercard, Carte Bancaire)
  \item 3D Secure renforcé conforme aux normes bancaires tunisiennes
  \item Webhook de confirmation avec vérification signature HMAC
  \item Statuts temps réel : autorisation, confirmation, annulation, remboursement
  \item Avantage : \textbf{coûts de transaction réduits} (interbancaire), confiance utilisateur maximale
\end{itemize}

\subsubsection*{Stripe (Paiements internationaux)}
\begin{itemize}[leftmargin=*,noitemsep]
  \item Intégration Stripe Elements (champs carte sécurisés)
  \item PaymentIntent + 3D Secure automatique
  \item Support cartes Visa/Mastercard/Amex
  \item Webhook confirmation + gestion remboursements
  \item Cible : touristes, expatriés, clients internationaux
\end{itemize}

\vspace{0.3cm}
\noindent\textbf{Sécurité :} Aucune donnée bancaire stockée côté serveur (PCI DSS compliant). Les deux gateways sont hébergées et sécurisées par leurs institutions respectives.

\subsection{Stratégie de Paiement}

Nous adoptons une approche \textbf{dual-gateway} pour maximiser le taux de conversion :
\begin{itemize}[leftmargin=*,noitemsep]
  \item Client tunisien → automatiquement redirigé vers \textbf{Click to Pay} (frais réduits, familiarité)
  \item Client international → automatiquement redirigé vers \textbf{Stripe} (acceptation globale)
  \item Détection automatique par géolocalisation IP + choix manuel disponible
  \item Unified backend : tous les paiements gérés par une API commune unifiée
\end{itemize}

\subsection{Avantages de Click to Pay (BCT)}

\begin{itemize}[leftmargin=*,noitemsep]
   \item \textbf{Frais de transaction avantageux} : taux préférentiels interbancaires (1.2--1.8\% vs solutions tierces)
   \item \textbf{Confiance maximale} : solution officielle de la Banque Centrale de Tunisie
  \item \textbf{Expérience fluide} : single-click payment pour clients enregistrés
  \item \textbf{Sécurité bancaire} : authentification forte 3D Secure obligatoire
  \item \textbf{Règlement rapide} : délais de virement réduits (24--48h vs 72h+)
  \item \textbf{Soutien à l'économie locale} : privilégier les solutions bancaires tunisiennes
\end{itemize}

\subsection{Back-office Administrateur}

Dashboard complet avec modules :
\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Véhicules} : CRUD complet, upload images multiples, changement statut (disponible/loué/maintenance)
  \item \textbf{Réservations} : Vue tableau + calendrier, filtres par statut/client/véhicule, actions (confirmer, annuler, marquer payé)
  \item \textbf{Clients} : Liste clients, historique, statut actif/inactif
  \item \textbf{Finances} : Revenus par période, gráphiques mensuels, exports CSV
  \item \textbf{Maintenance} : Suivi entretiens, alertes automatiques (next due date), documentation PDF
  \item \textbf{Tarifs} : Configuration règles (saison, weekend, promotions), application automatique
  \item \textbf{Config agence} : Informations contact, heures d'ouverture, politique d'annulation
\end{itemize}

\subsection{Notifications Automatiques}

Envoyées automatiquement via SMTP (Nodemailer + Handlebars) :
\begin{itemize}[leftmargin=*,noitemsep]
  \item Confirmation réservation (avec lien contrat PDF)
  \item Rappel 24h avant départ
  \item Confirmation paiement réussi
  \item Notification annulation (avec motif)
  \item Alerte maintenance (admin uniquement)
\end{itemize}

\section{Sécurité et Conformité}

\subsection{Protocoles de sécurité implémentés}

\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{HTTPS obligatoire} (Let's Encrypt SSL) — chiffrement TLS 1.3
  \item \textbf{Authentification JWT} : tokens court terme (15 min) + refresh tokens sécurisés (7 jours, stockés HttpOnly cookies)
  \item \textbf{Rate limiting} : protection anti-brute force (5 tentatives login / 15 min)
  \item \textbf{Validation DTO} : filtrage strict des entrées (whitelist), conversion automatique types
  \item \textbf{Headers de sécurité} : Helmet.js activé (XSS, CSRF, HSTS)
  \item \textbf{Backups automatiques} : dump PostgreSQL quotidien (cron)
  \item \textbf{Aucune donnée bancaire stockée} : seul ID transaction gateway conservé
  \item \textbf{Logs sécurisés} : rotation journalière, accès restreint aux logs
\end{itemize}

\subsection{Rôles et permissions}

Matrice fine des droits :
\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Visiteur} : consultation catalogue + disponibilités
  \item \textbf{Client} : réservation, paiement, historique, annulation (selon politique)
  \item \textbf{Admin} : accès total (flotte, clients, réservations, finances, maintenance)
\end{itemize}

\subsection{Conformité réglementaire}

\begin{itemize}[leftmargin=*,noitemsep]
  \item RGPD / Loi tunisienne protection données : consentement explicite, droit d'accès/rectification
  \item Conditions générales de location intégrées au contrat PDF
  \item Politique d'annulation personnalisable (48h gratuit, frais 30--50--100\% selon délai)
  \item Archivage numérique des contrats (10 ans recommandé)
  \item \textbf{Conformité bancaire} : intégration Click to Pay conforme aux normes de la Banque Centrale de Tunisie
  \item \textbf{Souveraineté financière} : paiements locaux traités entièrement dans l'écosystème bancaire tunisien
\end{itemize}
